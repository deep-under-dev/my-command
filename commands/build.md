# /build - 기능 만들기 (전체 플로우)

## 사용법
```
/build "기능 설명"
```

## 플로우
1. **탐색** - 관련 코드 파악
2. **계획** - 구현 단계 설계  
3. **검증 기준 설정** - 테스트/성공 조건 정의
4. **구현** - 단계별 코딩
5. **검증** - 테스트 실행

## 규칙
- 코딩 전에 항상 계획 먼저
- 각 단계마다 검증
- 큰 기능은 작은 단계로 분할
- 완료 후 /clear 권장

## 출력
```markdown
## 기능: [이름]

### 1. 탐색 결과
- 관련 파일: ...
- 기존 패턴: ...

### 2. 구현 계획
1. [ ] 단계 1
2. [ ] 단계 2

### 3. 검증 방법
- 테스트: ...
- 성공 조건: ...

### 4. 다음 액션
"계획 확인 후 /clear 하고 '단계 1 진행' 이라고 해주세요"
```
