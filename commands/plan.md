# /plan - 기능 계획 세우기

## 사용법
```
/plan "기능 설명"
```

## 동작
1. 요구사항 분석
2. 구현 단계 나누기
3. 파일 구조 설계
4. 테스트 계획

## 출력 형식
```markdown
## 목표
[한 줄 요약]

## 구현 단계
1. [ ] 단계 1
2. [ ] 단계 2
...

## 파일 변경
- `path/to/file.ts` - 설명

## 테스트
- [ ] 테스트 1
- [ ] 테스트 2

## 다음 액션
/clear 후 "단계 1 진행해줘" 라고 말하기
```

## 규칙
- 계획만 세우고 코드 작성 안 함
- 최대한 작은 단계로 나누기
- 각 단계는 독립적으로 실행 가능해야 함
