# 워크플로우 규칙

## 기본 원칙

### 1. 계획 먼저
- 코드 작성 전 항상 /plan 실행
- 계획 없이 바로 코딩 금지

### 2. 컨텍스트 관리
- 계획 완료 후 /clear
- 각 단계 완료 후 /clear 권장
- 토큰 절약 + 컨텍스트 오염 방지

### 3. 단계별 진행
- 한 번에 한 단계만 실행
- 단계 완료 확인 후 다음 진행
- 테스트 통과 필수

### 4. 토큰 최소화
- 불필요한 파일 읽기 금지
- 필요한 부분만 수정
- 긴 출력 대신 요약

## 플로우
```
/plan "기능" 
    ↓
[계획 확인]
    ↓
/clear
    ↓
/step 1
    ↓
[테스트 통과]
    ↓
/clear
    ↓
/step 2
    ↓
...
```
